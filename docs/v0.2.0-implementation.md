### 4. `internal/cmd/auth.go` — store identity on login

After successful cookie extraction:
1. Parse `twid` cookie from Chrome (format `u=<numeric_id>`) to get user ID
2. Call `GetUserByID(userID)` to resolve handle
3. Store both in credentials.json
4. Print `✓ Logged in as @handle`

If `twid` cookie not available, fall back to verify_credentials flow BUT also extract the numeric `id_str` from the v1.1 response (it returns `{"id_str": "...", "screen_name": "..."}`) and store it as UserID. This ensures `likes` and `mentions` commands always have a numeric user ID available, regardless of login method.

- REST v1.1 `/1.1/followers/list.json`: `{"users": [...], "next_cursor_str": "..."}`
